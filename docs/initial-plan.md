# 玄幻地球 on Minecraft 初版方案

## 1. 定位

`玄幻地球 on Minecraft` 不是传统“修仙附属”，也不是单纯魔法 mod。它本身提供可独立游玩的灵性资源、异常地貌、灵物生态和超自然工艺；安装《我的地球》后，再把更真实的矿物、地质、水文、植物和工业材料接入玄幻路线。

核心关键词：

- 独立基底：默认使用 MC 原版与本模组资源完成闭环；检测到《我的地球》后启用更丰富的现实材料映射。
- 玄幻叠层：灵气、灵脉、功法、异象、灵植、妖兽、秘境、阵法、炼丹、炼器、符箓。
- 玩法便利：复刻内容和体系，不复刻痛苦难度。
- 独立仓库：不污染 `earth_online` 本体，也不承担 `earth_human` 的人物难度模拟。
- 与魔幻分支互通：灵力与魔力统一显示为法力值，两条路线贡献直接相加。

## 2. 与其他分支的边界

### 《我的地球》

负责现实地球常见内容：

- 地质、矿床、岩石、土壤、水文。
- 冶金、化工、能源、电力、机械、建材、电子材料。
- MC 原版物品生态和 JEI/手册路线。

本模组只通过可选标签与公开接口联动这些材料，不把玄幻内容写回对方，也不把它作为前置。

### `earth_human`

预计独立制作，不是本分支附属。它负责：

- 人物属性、体质、营养、伤病、疲劳、寿命、精神。
- 真实生存难度和人体系统。

本分支未来只提供联动接口，例如灵气环境影响体质、丹药影响身体系统、秘境环境施加状态。

### 本分支不做

- 不做真实人体难度。
- 不强制玩家修行。
- 不把所有玄幻路线做成线性升级科技树。
- 不用玄幻解释替代现实材料体系。

## 3. 初版内容范围

### 世界与资源

- 灵气浓度：按维度、群系、高度、地质结构、水体、月相或局部异象计算。
- 灵脉：依托山脉、断层、矿脉、地下水、晶洞等现实地质结构生成。
- 灵泉：多见于泉眼、地下水、石灰岩/方解石路线、山地裂隙。
- 灵植：依托土壤、水分、光照和灵气浓度生长，不做食物保存难度。
- 灵矿：不是替代现实矿，而是现实矿物受灵性浸染后的特殊矿材。
- 异象区域：雷泽、寒潭、炎脉、阴地、药谷、晶窟等。

### 玩家工具

- 入门功法：引气诀，负责把环境灵气转化为统一法力值。
- 罗盘类：寻灵罗盘，指示附近灵气梯度或灵脉方向。
- 采样类：灵气试纸、灵泉瓶、灵土样本。
- 手册类：玄幻志，解释灵脉、灵植、阵法、炼丹、炼器路线。

### 工艺路线

- 炼丹：药材 + 灵泉 + 火候/炉器，产出短期增益或未来供 `earth_human` 使用的药物。
- 炼器：现实金属/矿物 + 灵性材料，制作法器前置件、强化材料、特殊工具。
- 符箓：纸、朱砂、金粉、灵墨等现实/玄幻材料结合，提供一次性效果。
- 阵法：以方块结构表达功能区，例如聚灵、净化、传送、守护、培育。
- 灵植培育：土壤、水文、灵气浓度影响产出，但不做高压农学难度。

## 4. 第一版 MVP

第一版不追求大而全，目标是形成一个能玩的“发现 -> 采集 -> 加工 -> 使用”闭环。

### 方块

- 灵脉节点：地下稀有节点，附近提高灵气浓度。
- 灵泉：水源变体或方块，瓶装后用于炼丹/阵法。
- 灵土：适合灵植生长，可由土壤样本和灵泉加工得到。
- 聚灵阵核心：小型阵法控制方块。
- 丹炉：炼丹容器，先做单方块 GUI，后续可升级多方块。
- 符案：制作符箓的工作台。

### 物品

- 寻灵罗盘。
- 灵泉瓶。
- 灵草、寒露草、赤阳花、阴石苔。
- 朱砂灵墨、符纸、基础符箓。
- 灵铁胚、灵铜胚、灵晶碎片。
- 初级丹药：回气丹、清心丹、辟毒丹。

### 系统

- 统一法力值：灵力和魔力等价为同一条法力值，玄幻只写玄幻贡献。
- 灵气浓度查询和 tooltip。
- 灵气浓度影响灵植生长。
- 丹炉配方：材料 + 灵泉 + 燃料/火候。
- 符案配方：符纸 + 灵墨 + 材料。
- JEI 分类：炼丹、符箓、阵法、灵材净化。
- 手册界面：玄幻志，和 `地球 Online` 手册区分。

## 5. 风格方向

视觉上不能全是现代工业风，也不能全是古风装饰。应按内容分层：

- 灵脉/矿物：真实矿物纹理 + 微弱发光裂隙。
- 灵泉/水系：自然水文 + 清透发光，不做卡通魔法水。
- 丹炉/符案：传统器具风，但保持 MC 可识别方块外观。
- 阵法：几何纹路、石材、金属嵌线、矿物节点，避免纯平面贴纸。
- 灵植：现实植物结构 + 轻微异化，不做过度发光塑料花。

贴图策略：

- 最终品质贴图优先 `image-2`。
- Python/Pillow 仅用于裁切、缩放、检查图集和临时占位。
- 首批先生成少量代表块：灵脉节点、灵泉、丹炉、符案、聚灵阵核心。

## 6. 技术初始方案

目标版本暂按 `NeoForge 26.2.0.7-beta / MC 26.2 / JDK 25`，与 `earth_online` 当前主线一致。

初始包名建议：

```text
com.xxsx.earthonline.xuanhuan
```

mod id 建议：

```text
earth_online_xuanhuan
```

模块结构建议：

```text
earth_online_xuanhuan/
├── docs/
├── neoforge-26.2/
│   ├── build.gradle
│   ├── settings.gradle
│   └── src/main/
└── README.md
```

依赖策略：

- 第一阶段可独立运行，不硬依赖 `earth_online`。
- 第二阶段添加软联动：如果检测到 `earth_online`，启用材料兼容、矿物联动和 JEI 路线扩展。
- 第三阶段与 `earth_human` 联动，但不把人物系统写进本 mod。

## 7. 第一轮开发顺序

1. 创建 NeoForge 26.2 工程骨架。
2. 注册基础物品、方块、创造模式标签和中英语言。
3. 实现灵气浓度基础系统：先用维度/群系/高度/方块邻域计算。
4. 实现寻灵罗盘和灵气 tooltip。
5. 实现灵脉节点、灵泉、灵土、基础灵植。
6. 实现丹炉和符案两个 GUI。
7. 添加 JEI 分类和玄幻志手册。
8. 部署到 26.2 NeoForge 实例测试。

## 8. 风险点

- 玄幻系统容易膨胀，必须先做少量闭环。
- 与 `earth_online` 联动不要变成硬依赖，否则独立仓库失去意义。
- 不要把 `earth_human` 的真实人体难度提前混进来。
- 世界生成仍要按 26.2 Codec 规则验证，不能只靠 JSON 语法检查。
