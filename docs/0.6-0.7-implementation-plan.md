# 玄幻地球 0.6-0.7 实施计划

## 1. 本轮交付版本

- 目标版本：`0.7.0-alpha.1`
- MC / Loader：Minecraft 26.2 / NeoForge 26.2.0.7-beta
- 部署策略：全部功能、本地双模测试和资源检查完成后一次性部署。
- 写入边界：仅修改 `xuanhuan_earth`；相邻项目只读。

## 2. 生物生态完成线

### 2.1 独立衍生实体

1. 灵化狐：探索型灵兽，已有基础实体，升级为可亲和/契约。
2. 风行狼：护卫型灵兽，提供追随、防卫与短时移速辅助。
3. 岩甲山羊：山地型灵兽，提供抗击退和寻矿环境反馈。

### 2.2 原创实体

1. 晶背灵龟：水岸/灵泉生态，防护与水下探索辅助。
2. 赤羽灵鹤：山林生态，缓降、远行和灵气感知辅助。
3. 修行聚落居民：散修商人、灵泉守护者、宗门执事三种职业变体。

每个实体必须拥有独立 `EntityType`、属性、AI、模型、动画、贴图、掉落、生成蛋或聚落生成来源，以及中英名称。不得修改任何原版实体。

## 3. 灵兽契约闭环

- 新增契约物品“灵兽契印”。
- 玩家先使用灵草提高亲和，再用契印完成契约；不能直接空手秒收服。
- 契约由服务端校验并写入实体所有者，物品消耗、粒子和提示同步。
- 已契约灵兽支持三种模式：跟随、驻守、自由活动。
- 不同灵兽提供不同近场辅助，但不直接覆盖玩家原版属性基值。
- 主人死亡、跨维度、断线和实体重载不能丢失所有权或产生复制。

## 4. 首批实体聚落

1. 散修坊市：摊位、茶棚、符材/药材交易，生成散修商人。
2. 灵泉村：灵泉池、药圃、仓房，生成灵泉守护者和少量灵化狐。
3. 宗门边境前哨：门楼、演武区、经卷房，生成宗门执事。

实现方式：

- 使用本模组独立 `Feature` 和数据配置生成，不替换原版村庄或结构。
- 聚落锚点方块保存类型、初始化状态和居民生成状态。
- 锚点首次加载后服务端生成持久居民，避免世界生成线程直接创建复杂实体。
- 居民使用原版商人容器交互，但交易表、职业、名称和产物属于本模组。
- 当前目录中的山灵祠、灵兽师营地保留为后续批次，不伪装成已生成内容。

## 5. 美术与资源

- 使用 Image-2 生成每种实体的材质母版，后台执行，不阻塞编码。
- 最终实体贴图由母版裁切为明确 UV，不把完整概念图贴到模型表面。
- 三类聚落锚点和居民视觉统一为东方木石、陶瓦、青玉与金属嵌线，但职业轮廓必须可区分。
- 最终分辨率检查、中英语言键检查、JSON 检查和资源引用检查均为发布门禁。

## 6. 验收标准

- 6 个实体均可通过命令生成并读取正确属性。
- 自然生成/聚落生成来源均成功注册，无未注册 SpawnPlacement 警告。
- 契约、模式切换、保存重载和主人判定通过服务端测试。
- 三类聚落可在新世界生成，锚点只初始化一次，居民可交易。
- 仅安装本模组时服务端到 `Done`，可从原版材料进入修行、设施和契约闭环。
- 与《魔幻地球》同时安装时服务端到 `Done`，共享法力不覆盖彼此数据。
- 客户端完成资源重载，日志无本模组引起的 `ERROR/FATAL`。
- 仅在以上全部通过后部署、推送 GitHub 和发布测试版。
