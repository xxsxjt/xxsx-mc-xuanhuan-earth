# 玄幻地球 0.7.0-alpha.4 功能质量契约

## 范围与边界

- 写入范围仅限本仓库；《我的地球》《地球人》和魔幻地球只读取公开契约。
- 模组必须可独立游玩；联动缺失时保留原版材料与原版玩家状态的完整回退路径。
- 未获“单独运行”授权时，不控制客户端；构建、资源和静态验证不得冒充实机玩法验证。

## 玩家闭环

| 阶段 | 权威状态 | 玩家表面 | 失败恢复 |
|---|---|---|---|
| 入门 | 手册不写修炼状态 | 泥土、木板或石头合成《玄幻地球手册》 | 手册首页给出第一步与下一步 |
| 解锁 | 功法等级从 0 变 1 | 《引气诀》及高级功法只负责首次学习 | 缺少前置时明确指出要先学哪一本 |
| 成长 | 各路线独立等级、XP、上限与迁移默认值 | 面板、蒲团 HUD、手册状态页 | 冷却、灵气不足和路线锁定都有恢复动作 |
| 长期收益 | 共享法力和身体适应 NBT | 显示法力、减耗、防护、呼吸倍率与当前等级 | 双模并装由玄幻模块统一消费共享身体适应，禁止重复结算 |
| 主动功法 | 服务端校验路线、法力和冷却 | 可配置快捷键、面板按钮、动作状态、效果与失败提示 | 重复包只会命中冷却，不重复扣费或结算 |

## 领域职责

- `XuanhuanHandbookItem`：教学、实时状态和下一步，不解锁、不修炼。
- `QiGuidingManualItem` / `CultivationManualItem`：一次性解锁路线，不恢复、不训练、不施法。
- `CultivationPractice`：环境读取、修炼冷却、XP、恢复与升级结算。
- `CultivationSkill`：即时主动功法。
- `ArcanaPower`：持久状态、迁移默认值、成长公式与共享状态契约。
- 被动事件层：只把游戏事件转交给成长收益计算，不拥有第二套等级或规则。

## 动作与生物

- 玩家修炼、恢复和战斗动作使用短生命周期同步状态；删除或消费写后不读的动作数据。
- 灵兽模型必须分别消费攻击进度、移动、坐守和物种状态；目标存在不能替代攻击时序。
- 狼/狐扑击，山羊顶撞，灵龟缩守，灵鹤展翼；不修改原版生物类或属性。
- 原版粒子、声音和药水仅作兼容反馈，不作为玄幻体系本身的全部实现。

## 教学与诊断

- 手册首页显示当前是否掌握引气、当前路线等级/XP、已解锁路线和下一步。
- 高级路线、灵气来源、坐具优势、主动功法与联动内容分开说明。
- 正常、未解锁、冷却、低灵气、满级和缺少可选模组状态都要有可执行提示。

## 验证矩阵

- JSON、语言 key、物品定义、模型与配方结构校验。
- 旧一级存档默认 `0 XP`，共享状态字段不因单模/双模切换重复加成。
- 快速重复修炼、重复技能包和未解锁请求由服务端拒绝。
- 离线构建、JAR 内容与部署哈希一致，实例中只保留一个有效同模组 JAR。
- 实机仍需单独授权验证：手册最小分辨率、快捷键、升级结算、被动收益和全部实体动作。
