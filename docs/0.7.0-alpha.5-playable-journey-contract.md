# 玄幻地球 on Minecraft 0.7.0-alpha.5 独立可玩旅程契约

## 目标

把现有功法、修炼、设施、主动功法、丹药/符箓和灵兽契约串成一条可被玩家理解、完成并复盘的生存旅程。不得用新增注册物掩盖断链，也不得要求安装《我的地球》《地球人》或魔幻模组。

## 六个核心里程碑

1. 研读《引气诀》，真正解锁周天路线。
2. 在面板或蒲团完成一次服务端结算的有效修炼。
3. 从丹炉、符案或聚灵阵的产物槽取出一次真实加工产物。
4. 成功施展一次当前路线主动功法并支付法力。
5. 成功使用回气丹或已充能基础符箓。
6. 完成一次灵兽亲和与契印契约。

里程碑写入玩家持久状态，只能由对应服务端成功路径触发。失败、冷却、法力不足、输出未取出和客户端点击都不得推进。

## 玩家表面

- 手册首页显示 `已完成/总数`、当前里程碑和可执行的下一步。
- 每个里程碑首次完成时只提示一次，不重复刷屏。
- 六项完成后给出一次轻量现有资源奖励和继续精通路线的提示；奖励只使用现有物品。
- 旧存档根据已学习和已有经验静默恢复可证明的步骤，不伪造设施、技能、消耗品或契约完成记录。

## 权威与兼容

- 状态保存在 `earth_online_arcana.xuanhuan_journey_mask`，由现有克隆数据所有权规则保留。
- 状态 Payload、Codec、空值、服务端构造和客户端手册必须同批升级。
- 自动化可以运行设施，但旅程的“亲手加工”里程碑只在玩家从输出槽取物时推进。
- 模组独立安装时全部六项可从原版材料完成；可选联动只能提供替代材料或恢复增强。

## 验证

1. 纯位掩码回归：顺序无关、重复幂等、完成数正确、下一步稳定。
2. 搜索六个完成钩子，确认都位于成功分支。
3. JSON、双语键、Payload 协议和完整构建通过。
4. 仅部署一个 alpha.5 jar，构建、实例与远端 Release SHA256 一致。
5. 未获“单独运行”授权时，不把静态与构建证据冒充实机完成。
