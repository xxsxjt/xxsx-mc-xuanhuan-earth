# 玄幻地球 0.7.0-alpha.3 逻辑审查与修复计划

## 1. 本轮目标

本轮不继续堆注册项，优先把已有内容从“能生成、能打开”修成可理解、可成长、可反馈的玩法闭环。写入范围仅限 `xuanhuan_earth`；相邻项目只用于读取公开联动契约。

运行验证仅做源码、资源、离线构建、JAR 与日志检查。除非用户明确说“单独运行”，不启动或控制 Minecraft、启动器、Blender 或其他桌面程序。

## 2. 已确认问题

1. `qi_guiding_manual` 同时承担新手指导书、功法、手册入口和自动修炼，职责互相冲突。
2. 泥土、木板、石头直接合成《引气诀》，实际把“免费新手指南”错误地变成“免费功法”。
3. `CultivationPractice` 会在玩家未学习功法时自动调用 `learnQiGuiding`，功法物品因此不是实际前置。
4. 修炼等级只在首次学习时写为 1；重复修炼没有经验、升级、成长反馈或等级收益。
5. 功法书右键既学习又恢复法力；学习物、修炼动作和主动技能没有分离。
6. 连续蒲团修炼会重复输出常规聊天信息，缺少安静结算和升级时的重点反馈。
7. `last_action` 只写入持久数据，没有客户端消费者，不构成动作系统。
8. 灵兽只有行走摆动和 `target != null` 的笼统姿态；攻击时序、坐姿与物种动作没有完整进入渲染状态。
9. 手册、JEI、tooltip、README 和元数据仍描述旧的混合职责与“自动掌握”行为。

## 3. 修复契约

### 3.1 物品职责

- `xuanhuan_handbook`：低门槛新手指导书，只打开图形手册并显示状态，不解锁功法、不执行修炼。
- `qi_guiding_manual`：基础功法，只负责首次学习；已学习后打开修炼面板。
- 辟谷、淬体、胎息功法：需要先掌握引气；首次使用解锁路线，之后选择对应路线并打开面板。
- 修炼面板/蒲团：唯一的重复修炼入口，负责环境读取、冷却、恢复、经验和升级。
- 主动功法：单独按钮与可配置快捷键触发，服务端校验法力、解锁和冷却。

### 3.2 成长闭环

每条路线具备独立等级、经验、下一级需求和上限。每轮有效修炼按灵气与辅助效率获得经验；升级时增加该路线可解释的法力、吸收、耐力、淬体、呼吸或辟谷收益。旧存档的一级数据继续有效，新增经验键默认从 0 开始。

### 3.3 动作闭环

- 玩家主动功法至少广播手臂动作、物种/路线化粒子与音效；结果仍由服务端判定。
- 灵兽渲染状态同步攻击进度与坐姿。
- 狼/狐表现扑击与伏身，山羊表现顶撞，灵龟表现缩守，灵鹤表现展翼。
- 动画不改变原版实体，也不依赖可选动画模组。

## 4. 静态验收

1. 指导书、功法、主动技能没有同一次右键多重执行。
2. 未学习基础功法不能通过面板或蒲团绕过前置。
3. 每次有效修炼都会推进对应经验，升级有持久增益和明确提示。
4. 面板显示当前路线等级、经验和主动技能冷却。
5. 实体模型消费真实攻击进度和坐姿状态，不再只看是否存在目标。
6. 中英语言、手册、JEI、配方、README、metadata 与实现一致。
7. 两个 NeoForge 26.2 模组离线构建通过，JAR 资源与源码一致，只部署当前有效版本。
