# 灵气 / 魔力场与冥想系统规划

## 当前实现状态

`xuanhuan_earth` `0.1.1` 已实现第一版运行骨架：

- 新增 `ArcanaChunkField`：按维度和区块缓存灵气场，基础值由维度、高度、世界种子和区块坐标稳定生成。
- 已接入来源扫描：`spirit_vein_node`、`spirit_spring_stone`、`spirit_soil`、`spirit_array_core`，以及丹炉/符案的弱结构加成。
- 已接入 depletion：引气诀恢复法力时消耗本区块灵气；枯竭度随游戏时间懒恢复。
- 已接入冷却：已学会引气诀后不能连续右键刷冥想恢复。
- 已接入玩家反馈：罗盘、引气诀和玄幻环境方块都会显示灵气等级、主要来源和区块枯竭度。

`fantasy_earth` `0.1.1` 也已接入同思路但不同语义的第一版运行骨架：

- 新增 `AetherChunkField`：按区块缓存以太场，使用自然背景、以太晶簇、仪式/附魔、符文/书册、炼金/能量材料作为来源。
- 已接入 disturbance：魔法调律恢复共享法力时会扰动本区块以太场，随后懒恢复。
- 已接入冷却：启蒙笔记与奥术适应札记共用调律冷却，避免换笔记连续刷法力。
- 已接入玩家反馈：魔幻手册、启蒙笔记、适应札记和魔幻方块都会显示以太等级、主要来源和扰动度。

仍未实现：跨区块扩散、源点持久登记、灵草成长事件、灵晶/灵石/魔盐/水晶消耗恢复、丹药冷却/丹毒、魔幻仪式多方块、手册 GUI 展示区域状态。

## 定位

玄幻的灵气和魔幻的魔力都接入统一法力值，但来源不同。

- 灵气：偏自然、地脉、灵植、灵泉、阵法，和区块环境强绑定。
- 魔力：偏以太晶体、仪式、符文、炼金、天象，也可参考同一套区块场模型。
- 法力值：玩家可用资源，仍使用 `earth_arcana.current_mana` 等共享键。

目标不是复刻修炼难度，而是让玩家能理解“为什么这里适合修炼、为什么灵草长得好、为什么魔法仪式更强”。

## 区块级场模型

不要继续把全部逻辑堆在一次性的 `Spirituality.measure(level, pos)` 里。第一版应改成区块级缓存：

```text
ArcanaChunkState
- dimension
- chunkPos
- baseQi
- veinQi
- floraQi
- structureQi
- depletion
- disturbance
- lastUpdateGameTime
```

内部数值建议用 `0..10000`，玩家显示成 `0..100`，避免小数抖动。

### 基础值

基础灵气由稳定因素决定，只在区块首次访问或缓存失效时计算：

- 维度：主世界正常，下界偏燥烈，末地偏虚空。
- 群系：山地、繁茂洞穴、森林、水源附近略高；荒漠、枯地略低。
- 高度：地下深层和山脊可有不同加成，但不要让 Y 值单独决定一切。
- 地质：如果安装《我的地球》并提供矿脉/岩性 tags，可给灵脉、晶体、富含水的岩层加权；否则使用本模组与原版环境判定。
- 噪声：用 world seed + dimension + chunk 坐标生成稳定小扰动，避免棋盘格。

### 来源值

来源值来自具体方块或世界生成结果：

- `spirit_vein_node`：强源，中心区块大幅提高，向周边 3-6 个区块衰减。
- `spirit_spring_stone`：水性灵气源，适合炼丹、灵草、恢复。
- `spirit_soil` / 灵草：灵植不是白嫖永动机。幼苗消耗少量灵气，成熟后可提高局部稳定度，收获会造成短期 depletion。
- `spirit_array_core`：结构性调节器，形成阵法后可把周围区块灵气缓慢汇聚到核心区块。
- 魔幻的 `aether_crystal_cluster`：对应以太源，给 magic/mana 场加权，而不是直接改 qi。
- 仪式台/炼金台/符文台：工作时消耗或扰动场，完成后有短暂回流。

## 更新算法

区块场只在必要时更新，不做全世界逐 tick 扫描。

1. 玩家进入或查询区块时，加载/创建 `ArcanaChunkState`。
2. 每个区块最多每 `20-60s` 游戏时间懒更新一次。
3. 更新时只计算本区块和少量邻居，做低成本扩散：
   - 高浓度向低浓度流动，系数小，例如 `0.03`。
   - 来源区块持续恢复。
   - depletion 随时间缓慢衰减。
4. 方块放置/破坏/世界生成时登记源点，不频繁扫描整个区块。
5. 如果登记表缺失，允许低频兜底扫描，例如玩家首次用罗盘查询该区块时扫描一次。

伪公式：

```text
raw = baseQi + veinQi + floraQi + structureQi - depletion - disturbance
display = clamp(raw / 100, 0, 100)
```

扩散：

```text
flow = (neighbor.raw - self.raw) * 0.03
self.raw += flow
neighbor.raw -= flow
```

## 冥想与吸收

冥想不能和丹药一样“右键就白回”。它应该是低成本但慢、有冷却、吃环境的位置型恢复。

### 引气诀 / 玄幻冥想

- 首次右键：学习功法，增加玄幻路线法力上限。
- 后续右键：进入冥想尝试。
- 冷却：建议 `30s` 起步，随功法提升下降到 `15s`，不能无间隔刷。
- 条件：玩家最好静止、未受伤、非饥饿濒危；第一版可以只做冷却。
- 恢复：由本区块灵气、玩家吸收率、是否在阵法/灵泉/灵脉附近决定。
- 消耗：恢复法力会给区块增加 depletion，浓度越低收益越小。

示例：

```text
restore = min(missingMana, 1.0 + localQi / 25.0 * absorbRate)
depletion += restore * 35
```

### 灵石 / 灵晶吸收

灵石吸收和冥想要有明显区别：

- 不强依赖本地灵气。
- 冷却短，例如 `5s`。
- 消耗物品或消耗耐久/充能。
- 恢复稳定，但成本更高。
- 低阶材料用 `spirit_crystal_shard`，后续加入完整灵石、灵核等。

### 丹药

丹药是即时恢复和状态型资源：

- 冷却中等，例如 `10-20s`。
- 可叠加药性/丹毒/耐受，避免无限吃。
- 不直接抽空区块灵气。
- 与冥想形成三角：冥想省材料但慢，灵石稳定但耗材，丹药即时但有药性代价。

## 魔幻侧参考

魔幻不是换皮灵气，应使用同架构不同权重：

- `aetherField`：以太晶簇、符文结构、天象、附魔台/书架等影响。
- 冥想叫“调律”或“聚焦”，恢复共享法力。
- 调律可消耗 `crystallized_mana_salt` 或 `aether_crystal` 作为焦点。
- 仪式结构可临时提高本区块 magic field，但会增加 disturbance。
- 魔幻和玄幻都练时，上限相加；恢复时分别按当前位置 qi/magic 场取高或组合。

## 玩家可理解的反馈

所有反馈先说人话，再给高级数值：

- “这里灵气稀薄，冥想几乎没有效果。去山地、洞穴灵脉或灵泉附近试试。”
- “这里灵气活跃，本次冥想恢复 4.2 法力。”
- “刚刚收获灵草，本区块灵气短时间下降。”
- “以太场紊乱，仪式台需要等待稳定。”

罗盘/手册应显示：

- 当前区块浓度等级。
- 最近一次变化原因：灵脉、灵泉、灵草、阵法、枯竭。
- 新手建议：去哪里、放什么、等多久。

## 实施阶段

1. `v0.2`：新增区块场缓存、罗盘查询、冥想冷却、depletion。
2. `v0.3`：灵脉/灵泉/灵草源点登记，灵草成长受区块灵气影响。
3. `v0.4`：魔幻 `aetherField` 接入同一接口，调律冷却和焦点消耗。
4. `v0.5`：聚灵阵/仪式结构修改区块场，手册 GUI 展示区域状态。
5. `v0.6`：与《我的地球》的矿脉/岩性/材料 tags 做可选增强联动。

## 验证标准

- 重启世界后同一区块基础浓度稳定。
- 放置/破坏灵脉相关方块后，罗盘读数变化可解释。
- 冥想有冷却，不能连续刷。
- 丹药、灵石、冥想三条恢复路径区别明确。
- 不出现全世界 tick 扫描导致卡顿。
- 魔幻和玄幻共享法力值，但环境来源和玩法语义不同。
