# 共享法力值与玄幻修行协议

## 核心决定

`玄幻地球` 的灵力不单独做一条孤立能量槽。灵力、《魔幻地球》的魔力和未来其他超自然路线的基础能量，在同时安装时统一显示为“法力值”。

玩家同时练玄幻与魔幻时，不互斥、不覆盖，直接加法计算：

```text
法力上限 = 基础法力
        + 玄幻路线贡献
        + 魔幻路线贡献
        + 装备贡献
        + 临时效果贡献
```

当前第一版默认：

- 基础法力：20
- 学会《引气诀》：玄幻路线贡献 +30
- 学会魔幻分支奥术启蒙：魔幻路线贡献 +30

所以两个都练后，第一版基础总上限为 80。

## 共享持久数据与所有权

两个 mod 都读写同一组玩家持久数据键：

```text
earth_arcana.current_mana
earth_arcana.base_mana
earth_arcana.xuanhuan_mana_bonus
earth_arcana.magic_mana_bonus
earth_arcana.equipment_mana_bonus
earth_arcana.temporary_mana_bonus
earth_arcana.qi_absorption_rate
earth_arcana.magic_attunement_rate
earth_arcana.cultivation_level
earth_arcana.magic_research_level
```

玄幻分支写入自身路线的等级、经验、吸收率与法力贡献，魔幻分支写入自身路线的数据。两个模组都可以读取共享法力总量，但不得清空或覆盖另一条路线。

双模并装时：

- 玄幻模块负责复制 `earth_arcana.*` 克隆数据并消费共享身体适应效果。
- 魔幻模块检测到玄幻模块后让出这两项职责，避免死亡重生复制两次或被动收益重复结算。
- 单独安装任一模块时，该模块自行保留共享数据并提供完整回退路径。

## 入门、解锁与修炼

三类物品和动作必须保持分离：

- `玄幻地球手册`：由一块泥土、任意木板或任意石头合成，只负责教学、实时状态和下一步，不解锁功法。
- `引气诀`：由书、紫水晶碎片和红石合成，首次右键只负责学习基础路线。
- 高级功法：要求先掌握引气诀，首次右键只负责解锁对应路线。
- 修炼面板与蒲团：重复修炼入口，负责环境读取、冷却、法力恢复、经验和升级。
- 主动功法：通过独立按钮或快捷键触发，服务端校验解锁、法力和冷却。

灵气由 `Spirituality.measure(...)` 读取当前环境：

- 主世界基础灵气高于其他维度。
- 地下、山地、高处和局部噪声会影响灵气。
- 灵脉、灵草、灵泉、阵法和可选地质联动通过同一测量入口贡献，不另建互相冲突的灵气值。

## 与魔幻分支联动

魔幻分支的魔力调律与玄幻分支的灵气吸收等价为同一条“法力值”。

设计上要保持：

- 玄幻叫“灵力路线贡献”，魔幻叫“魔力路线贡献”。
- UI 和手册对玩家说“二者都是法力值来源”。
- 如果另一个 mod 不存在，本 mod 仍可独立运行。
- 共享键是当前兼容协议；后续稳定后再抽取版本化公共 API，不能让独立游玩依赖第三个强制前置。

## 当前扩展方向

1. 手册、面板和 JEI 共用真实等级、经验、冷却、配方与结构谓词。
2. 功法、炼丹、符箓和阵法保持不同的操作与表现，不复用通用机器换皮。
3. 聚灵阵、灵脉与灵草按区块环境影响修炼，但不得绕过功法解锁和服务端校验。
4. 丹药主要提供即时恢复或有限增益；符箓制作成本高、激发成本低；主动功法消耗玩家法力。
5. 所有可选联动都提供缺失依赖时的原版材料路线和明确手册提示。
